# GitHub Publication Skill

## Purpose

Use this skill when preparing the repository for publication, cleanup, or contributor-friendly onboarding.

## What to Keep in Mind

- The repository should remain easy to understand for a beginner.
- Documentation should support both local development and GitHub readers.
- CI should be simple and reproducible.
- The project should feel polished without becoming heavy.

## Project-Specific Expectations

- `README.md` should explain the purpose, quickstart, and test command clearly.
- `CONTRIBUTING.md` should explain how to propose changes and validate them.
- `.github/` should hold lightweight automation and templates.
- `.claude/` should provide repo-specific AI workflow guidance.

## Publication Checklist

- README overview is understandable.
- License is present.
- Contribution guide exists.
- CI workflow is present and aligned with `pyproject.toml`.
- Repo-specific Claude instructions are in place.

## Success Condition

A first-time reader should be able to understand the project quickly, run it locally, and contribute without confusion.
